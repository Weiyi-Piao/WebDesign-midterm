What is the Object-oriented Programming

The term "object" is not means objects in the reality such as a box or a bird.
Every variable are all a object.
Each object follows the behavior and state defined by each specific class.

So, what is a class?

What is a class

According to the definition, a "class" is a software blueprint for implementing objects of a giving type.

We can see "class" everywhere.
Every variables belongs to a specific class.
For example, int, string, float, etc.

We can also create our own class.
For example, in Python, if you call the reserve word "class", we can then create a class with defined name. you can see an example in here.

Encapsulation Inheritance Polymorphism
The role of Encapsulation is to hide the implementation
details of an object from the clients of the object.

For user who are using an objcet from a class, we need not
understand all the details inside the program. We just
need to know how to call those "Methods" inside a class.
Questions like:
  • How does the Method worked?
  • What are those exact data needed for this program?
Are not necessarily to know for users.
In other words, Classes hides the details. (Abstraction) 		If there are two classes that is extremely similar to each other,
(for example,with only 10% of difference) we can make these
two classes as subclasses of a superclass.

With the superclass, we only need to consider the unique
informations inside those two subclasses. This makes the
program more readable and also made the user easier to fix
bugs they made.
